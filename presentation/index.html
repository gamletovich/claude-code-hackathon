Context

The Challenge

Northwind Logistics has run on the same Python order-processing script since 2017. It works — mostly. The engineers who built it are gone. The docs are a folder of Word files nobody opens. The board approved "modernization" without defining what that means.

What we inherited

One 80-line function doing five jobs. No tests. Three secrets in plain text. SQL injection on every customer lookup. Eight levels of nesting.

What we were asked for

A path to modernization that doesn't require a big-bang rewrite and doesn't break warehouse ops' daily order flows mid-extraction.

Our answer: strangler-fig extraction, characterization-first. Extract one service at a time. Pin behavior before touching a single line.

Archaeology

What We Found — Five Documented Flaws

1

Hardcoded Credentials

DB password, API key, and SMTP password embedded at module level. In production. In git history since 2019.

2

SQL Injection

f"SELECT * FROM customers WHERE id = '{customer_id}'" — user input straight into the query.

3

Global Mutable State

Three module-level dicts mutated mid-validation. No locking. No rollback. Race conditions under any concurrent load.

4

8-Level Nesting

The happy path sits at 8 levels of indentation. Each edge case requires reading the entire pyramid to understand.

5

Mixed Concerns

HTTP fraud check + inventory mutation + customer balance update + SMTP notification — all interleaved in one function.

Strategy

Our Approach — Strangler-Fig Extraction

We ranked every candidate seam by coupling, testability, and business risk. Order validation scored best on all three: it's a pure computation trapped in a stateful context.

1

Characterization tests

Pin the monolith's observable behavior — bugs included — before touching a single line. 9 scenarios × 2 implementations = 18 tests.

2

Extract OrderValidator + eval scorecard

Clean class, injected repos, typed result, zero globals. Eval confirms: legacy 20/100 → modern 100/100. CI enforces it on every commit.

3

The Fence — PreToolUse hook

Mechanical guardrail blocks any write to legacy/. Paired with a CLAUDE.md prompt. ADR-001 documents why each mechanism was chosen.

4

The Scouts — 5 parallel subagents

Fan-out risk analysis: one Claude call per candidate seam, each scoring coupling / coverage / data tangle / criticality. Ranked output compared to the human ADR.

Step 2 — The Extracted Service

Design principles

✓
Dependency injection — repos are passed in, never imported
✓
Early returns — max nesting depth 3, no pyramid
✓
Pure validation — reads only, never commits state
✓
Typed result — ValidationResult carries the full contract

Architecture

ADR-001 — Why This Seam First

We chose to extract

Order validation

Lowest coupling (returns a value, no DB writes of its own). Immediately testable. Eliminates the credential and injection flaws in one seam.

We chose NOT to extract (yet)

Inventory writes

Riskiest seam — 3 globals, no rollback. Wait until the characterization suite is trusted in CI.

Event bus / containerization

Right long-term shape. Adds risk before the seam is proven. ADR-002 decision.

Hook vs. prompt: The boundary enforcement is a CLAUDE.md prompt ("do not modify legacy/"), not a PreToolUse hook — the team is small enough that a prompt is sufficient. If the team grows past 4 engineers, a hook is the right escalation.

How We Built It

Claude Code — What Worked

Three-level CLAUDE.md

Root → legacy → modern. Each directory gets the context that fits it. Legacy says "do not modify." Modern says "no globals, max depth 3."

PreToolUse hook

.claude/hooks/guard_legacy.py blocks any write to legacy/ mechanically. Paired with a CLAUDE.md prompt. ADR-001 documents prompt vs. hook — when each is the right tool.

Eval harness as CI gate

score.py runs on every push via GitHub Actions. If anyone commits a global to modern/, the score drops and the pipeline fails — no human review needed.

Mixin test contract

OrderProcessorContract defines 9 tests. Both legacy and modern inherit them. Same assertions, different setUp — behavioral equivalence proven mechanically.

Fan-out subagents (The Scouts)

5 parallel Claude API calls, one per extraction candidate. Each independently scores coupling / coverage / data tangle / criticality. Coordinator ranks and compares to the human ADR.

Time saved

All 9 challenges completed: monolith + extraction + 18 tests + eval + ADR + runbook + hook + scouts + deck. ~2 hours of human direction.
